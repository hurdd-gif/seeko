# 404 Page (Scroll-Drawn "Off the Map") Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Also invoke superpowers:test-driven-development for every code task, and interface-craft + make-interfaces-feel-better + emil-design-eng during UI work.

**Goal:** Add a branded, scroll-scrubbed 404 page to the rr-app — giant `404` numerals that self-draw as the user scrolls, on the Paper light canvas, served by a catch-all route.

**Architecture:** A standalone React Router 7 catch-all route (`path: '*'`) renders a self-contained `NotFoundRoute` on the `.overview-light` Paper canvas. A tall (`220vh`) scroll track holds a sticky, viewport-tall stage; `motion`'s `useScroll().scrollYProgress` drives each numeral's `pathLength` (0→1) through `useTransform` windows so they draw left→right and un-draw on scroll-back. `useReducedMotion()` short-circuits to a fully-drawn static state.

**Tech Stack:** Vite, React Router 7 (`src/rr-app`), motion v12 (`motion/react`), Tailwind v4 + `--ov-*` Paper tokens, lightKit helpers, Vitest + Testing Library, Playwright (visual QA).

**Design doc:** `docs/plans/2026-06-22-404-page-design.md`

## Conventions (read before starting)

- Route file pattern: `src/rr-app/routes/<name>.tsx`, export `function XxxRoute()` (router entry) and a testable `function XxxContent()`.
- Router + nav inventory live in `src/rr-app/routes.tsx`. `routeInventory` is the single source of truth; `isNavRoute` hides `showInNav: false` entries from the sidebar.
- Tests: `src/rr-app/routes/__tests__/<name>.test.tsx`, `import { render, screen } from '@testing-library/react'`, `import { describe, expect, it } from 'vitest'`, wrap in `<MemoryRouter>` from `react-router`.
- `@/` alias resolves in rr-app (used widely, e.g. `@/lib/...`, `@/components/...`).
- Paper tokens: canvas `--ov-bg` (#eeeeee), text ladder `#111`/`#505050`/`#808080`/`#9a9a9a`, eyebrow `#808080`, lone accent azure `#0d7aff`. Primary button = `BTN_PRIMARY` from `@/components/dashboard/lightKit` (dark `#111` pill).
- Test command: `npm test` (Vitest). Single file: `npx vitest run src/rr-app/routes/__tests__/not-found.test.tsx`.

> NOTE on test scope (per migration-fidelity memory): a Vitest+jsdom test proves the component **mounts and wires links/roles correctly**, not its motion/scroll visuals. Scroll-draw and reduced-motion *behavior* are verified in Task 5 via Playwright, not brittle motion-value assertions.

### Task 3: Wire the scroll-scrubbed draw (motion)

**Files:**
- Modify: `src/rr-app/routes/not-found.tsx`

**Step 1: Replace the component body with the motion-driven version**

```tsx
// src/rr-app/routes/not-found.tsx
import { useRef } from 'react';
import { Link } from 'react-router';
import { motion, useScroll, useSpring, useTransform } from 'motion/react';
import { BTN_PRIMARY } from '@/components/dashboard/lightKit';

const NUMERAL_4 = 'M120 0 L40 130 L160 130 M120 0 L120 200';
const FOCUS_RING =
  'focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-[#0d7aff]/40 focus-visible:ring-offset-2 focus-visible:ring-offset-[var(--ov-bg)]';

export function NotFoundRoute() {
  return <NotFoundContent />;
}

export function NotFoundContent() {
  const trackRef = useRef<HTMLDivElement>(null);

  const { scrollYProgress } = useScroll({
    target: trackRef,
    offset: ['start start', 'end end'],
  });
  const progress = useSpring(scrollYProgress, { stiffness: 120, damping: 28, mass: 0.4 });

  const draw4L = useTransform(progress, [0.0, 0.4], [0, 1]);
  const draw0 = useTransform(progress, [0.2, 0.6], [0, 1]);
  const draw4R = useTransform(progress, [0.4, 0.8], [0, 1]);
  const copyOpacity = useTransform(progress, [0.62, 0.95], [0, 1]);
  const copyY = useTransform(progress, [0.62, 0.95], [14, 0]);
  const hintOpacity = useTransform(progress, [0.0, 0.12], [1, 0]);

  return (
    <div className="overview-light min-h-[220vh] bg-[var(--ov-bg)] text-[#111] antialiased">
      <div ref={trackRef} className="relative h-[220vh]">
        <div className="sticky top-0 flex h-screen flex-col items-center justify-center gap-7 px-6">
          <p className="font-mono text-[12px] font-medium tracking-[0.18em] text-[#808080]">
            ERROR · 404
          </p>

          <svg
            viewBox="0 0 620 240"
            role="img"
            aria-label="404"
            className="w-[min(86vw,560px)]"
            fill="none"
            stroke="#111"
            strokeWidth={8}
            strokeLinecap="round"
            strokeLinejoin="round"
          >
            <motion.path d={NUMERAL_4} transform="translate(40 20)" style={{ pathLength: draw4L }} vectorEffect="non-scaling-stroke" />
            <motion.ellipse cx="80" cy="100" rx="70" ry="100" transform="translate(230 20)" stroke="#0d7aff" style={{ pathLength: draw0 }} vectorEffect="non-scaling-stroke" />
            <motion.path d={NUMERAL_4} transform="translate(420 20)" style={{ pathLength: draw4R }} vectorEffect="non-scaling-stroke" />
          </svg>

          <motion.div
            className="flex flex-col items-center gap-4 text-center"
            style={{ opacity: copyOpacity, y: copyY }}
          >
            <h1 className="text-[clamp(22px,2.4vw,28px)] font-semibold text-[#111]">
              This page wandered off the map
            </h1>
            <p className="max-w-md text-[14px] leading-relaxed text-[#505050]">
              The page you’re looking for doesn’t exist or was moved. Let’s get you back on track.
            </p>
            <div className="mt-1 flex items-center gap-5">
              <Link to="/tasks" className={`${BTN_PRIMARY} ${FOCUS_RING} active:scale-[0.98]`}>
                Back to tasks
              </Link>
              <Link
                to="/docs"
                className={`text-[14px] font-medium text-[#505050] underline-offset-4 transition-colors hover:text-[#111] hover:underline ${FOCUS_RING}`}
              >
                Open docs
              </Link>
            </div>
          </motion.div>

          <motion.div
            aria-hidden
            className="pointer-events-none absolute bottom-10 flex flex-col items-center gap-1 text-[#9a9a9a]"
            style={{ opacity: hintOpacity }}
          >
            <span className="font-mono text-[11px] tracking-[0.16em]">SCROLL</span>
            <svg width="16" height="16" viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth="2" strokeLinecap="round" strokeLinejoin="round">
              <path d="M6 9l6 6 6-6" />
            </svg>
          </motion.div>
        </div>
      </div>
    </div>
  );
}
```

### Task 4: Reduced-motion + accessibility guard

**Files:**
- Modify: `src/rr-app/routes/not-found.tsx`

**Step 1: Add `useReducedMotion` and short-circuit to the drawn/visible state**

- Import: `import { motion, useReducedMotion, useScroll, useSpring, useTransform } from 'motion/react';`
- Inside the component, after the transforms:
```tsx
  const reduce = useReducedMotion();
  const pl4L = reduce ? 1 : draw4L;
  const pl0 = reduce ? 1 : draw0;
  const pl4R = reduce ? 1 : draw4R;
```
- Numeral styles: `style={{ pathLength: pl4L }}`, `style={{ pathLength: pl0 }}`, `style={{ pathLength: pl4R }}`.
- Copy block style: `style={{ opacity: reduce ? 1 : copyOpacity, y: reduce ? 0 : copyY }}`.
- Scroll hint style: `style={{ opacity: reduce ? 0 : hintOpacity }}`.

Result: with `prefers-reduced-motion: reduce`, numerals render fully drawn, copy + CTAs are immediately visible, scroll hint is hidden, and the page still scrolls (native) without any scrubbing.
